Scrumworks

  1. We use ScrumWorks for product, planning sprints… It's Simple and intuitive. You can export the product into an Excel document which contains all data about the Product. (LinkedIn3)
  2. I agree with Juncal, Scrumworks is very effective scrum tool. The web based part of the tool makes it attractive. Recently I managed a sprint with virtual team members, and thanks to Scrumworks, without which I couldn't have imagined. (LinkedIn3)
  3. I used Scrumworks mostly which is good to allow distributed team keep track of the essentials whenever they like. It can be a bit fiddly and unintuitive in places though. Excel is useful when dev effort is part of wider organisational project. It is more user friendly for non-dev resources and for reporting upstairs. (LinkedIn3)
  4. ScrumWorks Basic is not open source, but offers free licenses with registration. http://danube.com/scrumworks/ (LinkedIn2)
  5. I've used excel and scrumworks and IMO they are not in the same league. No spreadsheet is going to compare to a purpose built tool. (BTW, I'm not a MS hater and have no vested interest in Scrumworks) (LinkedIn2)
  6. Mike Cohn's - Alternative Release Burndown Chart. ScrumWorks provides a useful and intuitive agile management product that you should evaluate and consider before making your purchase decision. The agile tools available on the market today do many similar things, some better than others, but ScrumWorks offers a feature that distinguishes them from the rest: ScrumWorks appears to be one of the only, if not *the* only, product(s) to implement Mike Cohn's Alternate Release Burndown Chart. This chart is an excellent tool for estimating and projecting the release date/range for your agile project and ScrumWorks generates it for you! Mike Cohn - Alternative Release Burndown Chart http://www.mountaingoatsoftware.com/alt-releaseburndown ScrumWorks - Enhanced Product Burndown http://danube.com/docs/scrumworks/pro/latest/reports.html#prodburnenhanced. (userstories.com)
  7. ScrumWorks Pro works well for our distributed teams. We have development teams in multiple locations throughout the world and all are now successfully using ScrumWorks Pro. The combination of ease of use and comprehensive documentation have allowed our teams to easily make full use of the product. Our single ScrumWorks server provides more than adequate performance for these teams and is of little burden to our administrators. Project stakeholders are able to view team progress from a single location even when those teams are distributed. All in all, we are very happy with ScrumWorks and plan to continue to expand its use within our organization. (userstories.com)
  8. I first tried Scrum Works Basic and thought the interface made sense and was easy to use. When my company was ready to commit to an agile tool, I recommended we try Pro. Everyone was able to get up to speed in a week or so and we’ve never looked back. It helps new employees learn the Scrum process and, for veterans, it becomes an integrated part of the process. This is an awesome management tool. (userstories.com)
  9. Scrumworks : Excel avec Drag’N’Drop tout simplement. Pas compliqué, facile eh oui ne fait pas grand-chose mais fonctionne TRÈS bien! (focusintelligence.ca)
  10. I've used Scrumworks before which did a pretty good job, although if you're using TFS then Scrum for Team System is okay. (mobi)
  11. We use ScrumWorks and have no complaints! (mobi)

Against

  1. Quality Centre. Ok so I'm in HP and that influenced the decision somewhat but now that we're using it I find it very good. Used to use scrum works and it became a bit of an island not linked to any other systems. In QC requirements can be linked to tests (plans as well as execution instances) as can defects and/or incidents, it's web based and we're a virtual team distributed around the globe so that works well. Not trying to come across as an ad - a big install of QC needs plenty of care and feeding from an IT perspective so it is clearly not for everyone.but if you have it available then it might be worth investigating. (LinkedIn3)
  2. I've done the Scrumworks route, it left a lot to be desired, but got the basic job done. Recently working with www.rallysoftware.com which supports Jira integration as well, much better tool, the community edition is free, but not open source.
  3. Basically whiteboard + post-its , we did use Scrumworks in the past. We also have custom made cards for out planning poker sessions. (mobi)
  4. We tried RallyDev, VersionOne and Scrumworks. All of them are comprehensive tools but they they all took too long to set up and there were a lot of features we did not need. I went back to using Excel sheets as I was looking for something simple, till I came across ScrumEdge. (mobi)
  5. Ive previously used ScrumWorks and it worked mostly, but was rather flakey at times. (mobi)
